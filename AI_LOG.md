# AI Usage & Disclosure Log

**Project Component:** Authentication & User Role Routing (Login Interface)  
**AI Tool Used:** Gemini  

### 1. The Intent
I needed to implement a unified login system that routes three distinct types of users (Admin, PIC, and Applicant) to their respective dashboards based on a radio button selection, ensuring security and proper database verification.

### 2. The Interaction (Prompting)
I first asked Gemini for a basic login validation form. However, the initial response only covered a single generic user login. I refined the prompt to specify using radio buttons to divide Admin, PIC, and Applicant roles, asking for a backend logic structure that validates usernames and passwords based on the selected radio option.

### 3. The Output & Verification
Gemini provided the required frontend form code along with a conditional switch/if-else logic block for the backend. I tested the submission with mock credentials across all three options. The verification process caught an initial bug where submitting without a radio selection threw an error, which was manually resolved by adding client-side form validation attributes.

### 4. Human Refinement
I customized the redirection paths to match the exact names of my dashboard routes (`admin_dashboard`, `pic_dashboard`, and `applicant_dashboard`). I also added database query configurations to verify PIC credentials against the explicit table rows generated by the Admin registration module.

**Project Component:** Applicant Feature (Application Tracker & Deletion Logic)  
**AI Tool Used:** Gemini  

### 1. The Intent
I needed to program the state logic for the "Application Status" table. The business logic required that applicants can only delete their submitted job applications if the status is currently set to "Pending". If the status changes to "Approved" or "Rejected", the delete button must be programmatically disabled.

### 2. The Interaction (Prompting)
I requested Gemini to write a dynamic conditional rendering code block that checks a variable (`application_status`). I explicitly requested that if the status evaluates to "Approved", a specific text string must return: *"We will reach you soon via email/whatapps for an interview meeting"*.

### 3. The Output & Verification
Gemini successfully rendered the proper logic statement and included the exact notification string. I verified this functional behavior by updating dummy data rows in the database to toggle statuses between "Pending", "Approved", and "Rejected", confirming that the deletion workflow locked out exactly as intended.

### 4. Human Refinement
I integrated the logic into our system's dynamic user table views and custom-styled the approved message container to properly alert the user upon page load without breaking the layout grid.

**Project Component:** Admin Analytical Reporting (Monthly Visual Metrics)  
**AI Tool Used:** Gemini  

### 1. The Intent
I needed to implement an analytical feature in the Admin dashboard that aggregates incoming monthly application volume and generates a visual bar chart alongside a ranked textual list mapping total application trends from the highest down to the lowest volume.

### 2. The Interaction (Prompting)
I prompted Gemini to write a data-sorting algorithm that groups database timestamps by month and counts occurrences. I specified that the output array must sort descending based on company demands.

### 3. The Output & Verification
Gemini generated a data aggregation script and a compatible frontend configuration for a charting library. During data testing, some months with zero applications broke the sorting structure. I corrected this logic by writing a fallback clause to return a zero count instead of an empty return.

### 4. Human Refinement
I bound the generated array directly into our system's UI chart module and adjusted the list iteration variables to match the system's precise naming conventions for companies and job position data.

**Project Component:** Backend Database Integration & Error Handling (PIC Registration)
**AI Tool Used:** Gemini  

### 1. The Intent
I had already written the SQL query and backend logic to handle the Admin registering a new PIC profile (storing the company name, address, email, and credentials). However, the form submission kept failing with a database connection error/constraint violation, and I needed to find and fix the bug in my code.

### 2. The Interaction (Prompting)
I pasted my raw backend code block into Gemini along with the specific database error message from the console. I asked Gemini: *"My PIC registration form is throwing this specific error when I click submit. Review my query and data parsing logic to find out what is causing the crash."*

### 3. The Output & Verification
Gemini analyzed my code and pointed out that I was passing empty strings for optional fields (like `company_phone_number`) into a database column that was strictly configured as `NOT NULL`. It provided the corrected syntax to handle null values securely. I applied the fix and verified it by submitting multiple test registrations in the Admin dashboard, confirming that the PIC credentials now saved perfectly without crashes.

### 4. Human Refinement
Once the core bug was fixed, I added custom try-catch validation blocks around the code to ensure that if a duplicate username or company email is entered, the system safely triggers a user-friendly alert message instead of breaking the connection.
